PDF RuleFinder

The PDF RuleFinder is a simple Node.js-based utility that is used to find the page in a set of pdf files that best matches a set of keywords. It is the first non-trivial utility in my Dungeon-Master-Utility set, though it is not strictly limited to tabletop gaming use. I actually tested it using technical manuals after I realized that I do not actually have any PDF files containing tabletop game rules.

In short, it is a tool for searching through several PDF files at once to try to help the user find where to re-visit information.

This project is actually one of a broader set of tabletop gaming utilities that I am working on.

GitHub: https://github.com/spopkin/Dungeon-Master-Toolkit

List of Technologies and Techniques Used:

  • HTML 5
  • CSS 3
  • Node.js
  • JavaScript
  • JQuery
  • MongoDB
  • REST
  • Express
  • pdf-text-extract library
  • Wait.for fibers library

How it Works

When started with the --refresh-db flag, it uses pdf-text-extract to extract the text from all of the PDF files in a preconfigured rulebook directory. After the text is extracted, it is inserted into entries for each given book in MongoDB. Later, when the function to search is invoked, it searches through those extracted text entries and ranks them in terms of which page is the most likely to correspond to the searched-for keywords.

The frontend is an HTML, CSS, and JavaScript page that uses JQuery to retrieve information from the NodeJS REST API.

Design Decisions

  • It is a Node.js webapp to simplify future integration with other tools that I plan to make
  • It uses MongoDB, as the data used is just a bunch of documents
  • It preloads the text data for performance reasons

Why Do It?

I plan to get back into playing Dungeons and Dragons at some point. When I do, I will want to build my own campaign. That is a time consuming process that I would like to streamline. Creating a working DM's toolkit is one potentially effective means to do so.
